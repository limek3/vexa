# ClickBook — booking metrics, today page, visit status repair

## Что исправлено

### 1. Статусы записей
- Добавлен статус `no_show` для случаев, когда клиент не пришёл.
- Логика статусов теперь такая:
  - `new` — новая заявка, мастер ещё не подтвердил.
  - `confirmed` — запись подтверждена, ждём визит.
  - `completed` — клиент пришёл.
  - `no_show` — клиент не пришёл.
  - `cancelled` — запись отменена.

### 2. Записи на сегодня
- Страница `/dashboard/today` теперь показывает только записи на текущую дату.
- Бейдж в боковом меню рядом с “Записи на сегодня” считает только сегодняшние новые заявки, а не все будущие записи.
- Если на сегодня пусто, но есть будущие записи, страница показывает ближайшие будущие записи отдельным fallback-блоком.
- В карточке записи добавлены действия:
  - “Подтвердить” для новой заявки.
  - “Клиент пришёл”.
  - “Клиент не пришёл”.

### 3. Проверка прихода клиента через Telegram
- Добавлена серверная отправка мастеру вопроса “Клиент пришёл?” после окончания услуги + 15 минут.
- В сообщении мастеру есть inline-кнопки:
  - “Да, пришёл” → запись получает статус `completed`.
  - “Нет, не пришёл” → запись получает статус `no_show`.
- Webhook Telegram теперь обрабатывает эти кнопки и синхронизирует статус в `sloty_bookings` и JSON workspace.

Важно: на Vercel Hobby cron стоит 1 раз в день из-за ограничения тарифа. Для точного вопроса через 15 минут нужен внешний cron на `/api/cron/reminders` каждые 5–10 минут или Vercel Pro.

### 4. Статистика и аналитика
- Убраны верхние служебные плашки, которые мешали экрану.
- Метрики дохода, среднего чека, записей, каналов и дневного графика пересчитываются из реальных бронирований.
- График 30 дней теперь умеет включать будущие даты, если записи стоят на ближайшие дни.
- Источники нормализованы только в:
  - ТГ
  - Инстаграм
  - ВК
- Старые MAX/макс значения переводятся в ВК.

### 5. База данных
Добавлена миграция:

`supabase/migrations/20260501_0013_clickbook_booking_status_analytics_repair.sql`

Она добавляет поля для статусов, источников, цены, длительности и отметки отправленного запроса мастеру.
