# ClickBook repair: bot, chats, clients, services, mobile

Что исправлено:

1. Mini App из Telegram
- Добавлен watchdog на экране `/app`, чтобы Mini App не висел бесконечно на «Входим через Telegram».
- Если Telegram долго не отдаёт `initData`, приложение пробует открыть кабинет по сохранённой app-session и показывает прямую кнопку «Открыть кабинет».

2. Сообщения клиенту через бота
- Исходящие сообщения из `/dashboard/chats` теперь пытаются доставляться клиенту через Telegram bot, если клиент подключил Telegram через кнопку «Получить в Telegram» после записи.
- Настройка зависит от уведомления `chat-message` / «Сообщение клиенту» на странице «Уведомления».
- Добавлена обратная сторона: если клиент отвечает боту обычным сообщением, оно сохраняется в чат мастера как сообщение клиента.

3. Подтверждение записи клиенту
- После перехода клиента по booking deep-link бот сохраняет `chat_id` клиента, подтверждает связь с записью и обновляет чат.
- Клиентское подтверждение зависит от уведомления `visit-reminder` / «Напоминание клиенту».

4. Записи, клиенты и чаты
- Fallback-чаты из записей теперь при первом сообщении превращаются в реальные chat threads в Supabase, а не ломаются на временных id.
- Записи из `sloty_bookings` и JSON workspace остаются связаны с CRM, клиентами и чатами.

5. Заметки и напоминания в карточке клиента
- В CRM-карточке клиента добавлено редактирование заметки.
- Добавлено сохранение напоминания: текст + дата/время.
- Данные хранятся в `workspace.data.clientNotes` и `workspace.data.clientReminders`.

6. Услуги
- Исправлена причина «самоудаления» услуг: создание/обновление профиля больше не выкидывает пользовательские услуги, которых нет в seed-профиле.
- Защита от случайной отправки пустого массива услуг со старого snapshot.
- Если live-каталог пустой, страница услуг восстанавливает услуги из профиля/seed.

7. Уведомления
- Добавлен новый тип уведомления `chat-message` / «Сообщение клиенту».
- Старые сохранённые настройки уведомлений автоматически дополняются отсутствующими seed-уведомлениями.

8. Мобильная адаптация
- Добавлен компактный mobile override для dashboard-grid: карточки и метрики на телефоне стараются становиться 2x2, без поломки календаря и таблиц.

SQL:
- Добавлена миграция `supabase/migrations/20260501_0011_clickbook_notifications_clients_repair.sql`.
- Она безопасная и повторяемая: добавляет недостающие поля для Telegram booking links и chat metadata, а также расширяет delivery_state.
