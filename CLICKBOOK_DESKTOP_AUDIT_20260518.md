# ClickBook Desktop audit — 2026-05-18

## Кратко
Проект — Next.js + React веб-приложение с Electron shell. Desktop-версия живёт в `/desktop/*` и открывается внутри frameless Electron-окна. Текущий desktop-интерфейс функциональный, но собран как прототип: большая часть логики, UI, модалок и данных находится в одном файле `app/desktop/_components/desktop-workspace.tsx`.

## Что уже поправлено в этом архиве
1. Desktop-навигация больше не принудительно добавляет `?demo=1` на каждую страницу. Теперь query-параметры текущего режима сохраняются. Это важно для `desktop:live`, иначе live-кабинет после клика по меню возвращался в demo.
2. Модалка создания записи теперь реально сохраняет выбранную дату. Раньше поле даты отображалось, но значение не передавалось в `addBooking`.
3. Убран двойной вертикальный отступ под Electron titlebar: body уже резервирует высоту шапки, поэтому spacer не должен добавлять вторые 40px.

## Главные проблемы для следующего прохода
1. Убрать `// @ts-nocheck` из desktop-workspace и разнести файл на typed modules: shell, hooks, modals, pages, primitives.
2. Привести desktop UI к единой дизайн-системе: сейчас много экранов сделано рабочими, но часть блоков выглядит как демо-панели.
3. Сделать нормальную desktop-оболочку: сохранение размера окна, offline/read-only режим, экран ошибки авторизации, auto-update стратегия.
4. Переписать расписание как настоящий production calendar: drag/drop, создание по клику в свободный слот, пересечения, перерывы, блокировки.
5. Доработать CRM и чаты: связать клиента, запись, историю визитов, шаблоны и каналы в один сценарий.
6. Включить строгую проверку сборки: сейчас в `next.config.js` стоит `ignoreBuildErrors: true`, а главный desktop-файл отключает TypeScript.

## Ограничение проверки
В архиве нет `node_modules`, поэтому `npm run lint` не запускается без установки зависимостей. Electron main/preload проверены через `node --check`.
