# ClickBook Mini App — mobile strong patch

Что сделано в этом архиве:

- Усилена мобильная точка входа `/app`: на телефоне открывается Mini App, на десктопе — рабочий кабинет.
- Mini App теперь не зависает вне Telegram-окружения: Telegram initData авторизация запускается только когда реально есть runtime Telegram WebApp.
- Исправлена загрузка и отправка чатов:
  - чтение реальных `threads` из `/api/chats`;
  - корректная нормализация сообщений `master/client/system`;
  - создание нового чата отправляет `type: "thread"`, `clientName`, `clientPhone`, `channel`;
  - отправка сообщения отправляет `type: "message"`, `threadId`, `body`, `author: "master"`.
- Исправлено сохранение услуг:
  - услуги Mini App сохраняются в workspace section `services`, который читает публичная страница мастера;
  - старый `serviceCatalog` оставлен только как fallback для чтения старых данных;
  - текстовое описание профиля синхронизируется из активных видимых услуг.
- Исправлен график/слоты:
  - дни недели сохраняются с `weekdayIndex`, статусом, перерывами и рабочими диапазонами;
  - слоты теперь сохраняются как диапазоны `10:00–11:00`, которые понимает публичная форма записи;
  - перерывы не попадают в доступные слоты;
  - копирование понедельника переносит расписание, статус и перерывы.
- Улучшена мобильная устойчивость: данные обновляются через существующий workspace store, без отдельной разорванной логики.

Проверка:

- `node _syntax_check_local.js components/mini/mini-app-entry.tsx app/app/page.tsx` — OK.
- Полный `next build` в распакованном архиве не запускался, потому что в архиве нет `node_modules`. После установки зависимостей можно проверить командой `pnpm install && pnpm build`.
