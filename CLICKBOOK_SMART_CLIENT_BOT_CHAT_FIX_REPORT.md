# ClickBook stable auth + slots final fix

Исправлено по текущим проблемам:

1. Сессия после обновления страницы
   - `/api/auth/telegram/status` теперь возвращает `appSessionToken`, а не только HttpOnly cookie.
   - `TelegramLoginButton` сохраняет этот токен в `localStorage`.
   - Все защищённые запросы уже отправляют `X-ClickBook-App-Session`, поэтому Telegram/WebView/браузер больше не зависит только от cookie.

2. "Сессия истекла" при сохранении услуг/графика
   - `useWorkspaceSection` переписан на очередь сохранения.
   - Быстрые клики по слотам/услугам больше не создают гонку PATCH-запросов.
   - Старый ответ API больше не перетирает новый локальный выбор.

3. Слоты резко менялись сами по себе
   - Убран full-snapshot overwrite после сохранения section.
   - Локальное состояние графика теперь остаётся главным, пока save не завершён.
   - При быстром переходе на публичную страницу последний выбранный график всё равно уходит на сервер.

4. `/m/[slug]` не видел свежие слоты
   - Публичная страница для владельца теперь сначала берёт актуальный локальный `workspaceData`, если он есть.
   - API `/api/public/[slug]` больше не даёт старым нормализованным строкам `sloty_availability_days` перекрывать свежий JSON в `sloty_workspaces.data.availability`.
   - `/api/bookings` проверяет слоты в том же порядке приоритета: seed -> normalized fallback -> актуальный workspace JSON.

5. Мелкие фиксы
   - Исправлен случайный дубликат `const normalized` в `app/dashboard/availability/page.tsx`.

SQL не менялся: новые таблицы/колонки не требуются, если уже прогнаны миграции `0009` и `0010`.
