# ClickBook — status, analytics and today schedule repair

Что сделано в этой сборке:

1. Убрана лишняя механика ручного подтверждения записи.
   - Новая заявка из публичной формы теперь сразу создаётся со статусом `confirmed` / «Запланирована».
   - Логика такая: мастер сам открыл слот в графике, значит запись на этот слот уже допустима.
   - В кабинете оставлены только реальные действия после визита: «Клиент пришёл» и «Клиент не пришёл».

2. Статусы стали понятнее.
   - `confirmed` теперь показывается как «Запланирована», а не «Подтверждена».
   - Старые `new` тоже отображаются как «Запланирована», чтобы не было ощущения, что мастер должен что-то дополнительно подтверждать.
   - `completed` — «Пришёл».
   - `no_show` — «Не пришёл».
   - `cancelled` — «Отменена».

3. Доход стал фактическим.
   - Доход / факт считается только по записям со статусом `completed`.
   - Будущие и запланированные записи больше не попадают в фактический доход.
   - Средний чек считается только по завершённым визитам.

4. Записи на сегодня.
   - Если сегодня записей нет, экран показывает ближайшие будущие записи нормальным списком.
   - Правая карточка теперь открывается и для будущих записей, а не только для сегодняшних.
   - Бейдж в меню «Записи на сегодня» считает только записи сегодняшнего дня.

5. Аналитика и графики.
   - Графики и метрики берут данные из реальных бронирований.
   - Доход на графике — только факт по пришедшим клиентам.
   - Записи/источники/клиенты считаются по реальной базе.

6. Шаблоны.
   - Убрана старая логика «подтвердить запись» в текстах.
   - Добавлены корректные ссылки вида `https://www.кликбук.рф/m/{{slug}}`.
   - Источники/каналы нормализованы: ТГ, Инстаграм, ВК. MAX сворачивается в ВК.

7. SQL.
   - Добавлена миграция `20260501_0014_clickbook_status_revenue_templates_repair.sql`.
   - Она переводит старые `new`-записи в `confirmed`, проставляет `confirmed_at`, нормализует MAX → ВК и обновляет старые ссылки в шаблонах.

Проверка в этом окружении:
- ZIP собран и проверен на целостность.
- Полный `next build` не запускался, потому что в окружении нет `node_modules`.
