# КликБук desktop: объединение сайта и приложения

Сделан цельный desktop-проход: приложение больше не держится на старом разъезжающемся прототипе, а собрано как единый рабочий кабинет с навигацией по зонам, внешним видом из нового референса и функциональными страницами.

## Что изменено

### 1. Новый desktop shell
- `app/desktop/_components/desktop-workspace.tsx` переписан как стабильный client-only рабочий кабинет.
- Убраны зависимости, которые давали hydration mismatch и циклические setState.
- Все данные desktop хранятся стабильно через `localStorage`, но первый рендер всегда одинаковый — без Hydration failed.
- Все пункты меню используют реальные Next routes и `prefetch={false}`.

### 2. Меню снова разделено на зоны
- Рабочее место
- Витрина и запись
- Коммуникации
- Аналитика и доступ
- Настройки

### 3. Добавлен функционал сайта в desktop
Добавлены недостающие desktop-разделы:
- `/desktop/finance`
- `/desktop/marketing`
- `/desktop/payments`
- `/desktop/limits`
- `/desktop/sources`

Эти разделы добавлены в меню и в `DesktopScreenApp`.

### 4. Все текущие desktop-страницы сделаны функциональными
- Главная: KPI, записи, следующая запись, внимание, каналы, услуги.
- Расписание: дни, таймлайн, список, перерывы, создание записи.
- Клиенты: поиск, сегменты, профиль, заметки, избранное, экспорт CSV.
- Чаты: список диалогов, отправка сообщений, создание записи из чата.
- Услуги: редактирование названия, цены, длительности, видимости, дубль/удаление.
- График: рабочие дни, окна, перерывы, выходные.
- Профиль: публичная страница, контакты, описание, ссылка.
- Шаблоны: редактирование сообщений, включение/выключение, тест.
- Уведомления: включение/выключение событий.
- Интеграции: подключение/отключение каналов.
- Аналитика: KPI, каналы, услуги, динамика.
- Отзывы: редактирование и добавление отзывов.
- Тариф: планы и функции.
- Финансы: выручка, расходы, прибыль, оплаты.
- Маркетинг: возврат клиентов, свободные окна, отзывы после визита.
- Платежи: история оплат.
- Лимиты: использование тарифа.
- Источники: каналы заявок.
- Внешний вид: light/dark, акцент, плотность, сброс демо.

### 5. Новый визуальный слой
- `app/desktop/desktop.css` переписан под аккуратную SaaS desktop-систему.
- Взята логика из референса: строгий sidebar, разделы меню, чистые карточки, мягкие границы, аккуратные таблицы, нормальные состояния hover/active.
- Поддерживаются light/dark темы и разные акценты.

## Проверка

Синтаксис новых `.tsx` файлов проверен через TypeScript `transpileModule` без ошибок.
Полный `next build` не запускался, потому что в архиве нет `node_modules`, а установка зависимостей в sandbox недоступна.

После распаковки локально лучше запускать так:

```bash
rm -rf .next
npm install
npm run desktop:dev
```

Если Turbopack держит старый чанк — обязательно удалить `.next` перед запуском.
